## Privacy Audit: The Honest Scope

Before the code, the audit question answered directly.

**Q: Does anything in the current architecture violate the privacy promise?**

**A: No. But the promise requires precise language to be audit-proof.**

**Data that NEVER reaches any server:**

| Data | Storage | Server-accessible |
|------|---------|-------------------|
| Mood logs | SQLite (device) | Never |
| Journal entries | SQLite (device) | Never |
| Circadian history | SQLite (device) | Never |
| Sobriety start date | AsyncStorage (device) | Never |
| Restlessness score | SQLite column (device) | Never |
| Backup passphrase | Not stored | Never |

**Data the sponsor relay holds in memory (intentional, opt-in):**

| Field | Nature |
|-------|--------|
| Check-in date string | Presence signal ("Mon Mar 28 2026") |
| Orchid stage label | Milestone phase ("seedling") |
| Milestone count | Integer |
| Presence signals | type: water or light only |

These exist because the sponsor channel is designed to share "I showed up today" with one trusted person. No mood score, no journal content, and no sobriety start date is ever sent to the server.

**The corrected claim:** *"Your mood logs, journal entries, and all health-sensitive data physically cannot leave this device."* This is provably true.

## System 1: Database Initialization
**File:** `utils/db.ts` | 65 lines

```typescript
import { openDatabaseSync, type SQLiteDatabase } from 'expo-sqlite';
import { Platform } from 'react-native';

export const SCHEMA_VERSION = 1;

let _db: SQLiteDatabase | null = null;

export function getDb(): SQLiteDatabase {
  if (_db) return _db;
  if (Platform.OS === 'web') {
    throw new Error('SQLite unavailable on web');
  }
  _db = openDatabaseSync('kataleya.db', { enableChangeListener: false });
  _db.execSync(`PRAGMA journal_mode = WAL;`);
  _db.execSync(`PRAGMA foreign_keys = ON;`);
  migrate(_db);
  return _db;
}

function migrate(db: SQLiteDatabase): void {
  const current = db.getFirstSync<{ user_version: number }>('PRAGMA user_version')?.user_version ?? 0;
  if (current >= SCHEMA_VERSION) return;

  db.runSync(`
    CREATE TABLE IF NOT EXISTS mood_logs (
      id               TEXT PRIMARY KEY NOT NULL,
      ts               INTEGER NOT NULL,
      mood_score       INTEGER NOT NULL CHECK (mood_score BETWEEN 1 AND 10),
      context          TEXT,
      circadian_phase  TEXT NOT NULL,
      restlessness     REAL NOT NULL DEFAULT 0 CHECK (restlessness >= 0 AND restlessness <= 1),
      correction       TEXT
    )
  `);
  db.runSync(`CREATE INDEX IF NOT EXISTS idx_mood_ts ON mood_logs (ts DESC)`);

  db.runSync(`
    CREATE TABLE IF NOT EXISTS journal_entries (
      id               TEXT PRIMARY KEY NOT NULL,
      ts               INTEGER NOT NULL,
      body             TEXT NOT NULL,
      mood_score       INTEGER CHECK (mood_score BETWEEN 1 AND 10),
      circadian_phase  TEXT NOT NULL
    )
  `);
  db.runSync(`CREATE INDEX IF NOT EXISTS idx_journal_ts ON journal_entries (ts DESC)`);

  db.runSync(`
    CREATE TABLE IF NOT EXISTS circadian_log (
      id     INTEGER PRIMARY KEY AUTOINCREMENT,
      ts     INTEGER NOT NULL,
      phase  TEXT NOT NULL,
      event  TEXT NOT NULL
    )
  `);
  db.runSync(`CREATE INDEX IF NOT EXISTS idx_circ_ts ON circadian_log (ts DESC)`);

  db.runSync(`PRAGMA user_version = ${SCHEMA_VERSION}`);
}
```

**What to verify:**
- `PRAGMA user_version` migration guard — schema changes run exactly once, future versions increment the version constant safely
- `journal_mode = WAL` — non-blocking reads during writes
- `CHECK (mood_score BETWEEN 1 AND 10)` — rejected at database level, not application level
- No `user_id` column exists in any table — the database has no concept of identity

## System 3: Circadian Engine
**File:** `hooks/useCircadian.ts` | 68 lines

```typescript
export function useCircadian(): CircadianState {
  const [state, setState] = useState<{ phase: CircadianPhase; blend: number }>(computeState);
  const timerRef = useRef<ReturnType<typeof setTimeout> | null>(null);

  const schedule = useCallback(() => {
    const next = computeState();
    setState(next);

    const minutes = getCurrentMinutes();
    const inTransition = isInTransition(minutes);
    const delayMs = inTransition ? 10_000 : msUntilNextMinute();

    timerRef.current = setTimeout(schedule, delayMs);
  }, []);

  useEffect(() => {
    schedule();
    return () => { if (timerRef.current) clearTimeout(timerRef.current); };
  }, [schedule]);

  const theme = interpolateTheme(MorningBloom, MidnightGarden, state.blend);
  return { phase: state.phase, blend: state.blend, theme,
           phaseConfig: CIRCADIAN_PHASES[state.phase], isGoldenHour: state.phase === 'goldenHour' };
}
```

**What to verify:** No `fetch()` call. Phase computed from `getCurrentMinutes()` — a local clock read. The adaptive polling (10s during transition, 1-minute boundary otherwise) reduces JS thread wakeups during stable periods.

**Confidence formula:**
```typescript
function scoreConfidence(n: number, mostRecentTs: number): number {
  if (n < 3) return 0;
  const sizeScore = 1 - 1 / (1 + n / 12);
  const ageDays = (Date.now() - mostRecentTs) / 86_400_000;
  const recency = Math.exp(-ageDays / 45);
  return Math.min(0.97, sizeScore * recency);
}
```

## System 6: Zero-Knowledge Encrypted Backup
**File:** `utils/backup.ts` | 306 lines

**v2.2 upgrade:** The crypto backend was migrated from `crypto-js` (pure JavaScript) to `react-native-quick-crypto` (JSI native module). PBKDF2 key derivation runs in native OpenSSL (Android) / CommonCrypto (iOS). The OS CSPRNG provides entropy. `timingSafeEqual()` replaces string comparison for MAC verification — eliminating the timing side-channel. The blob format is unchanged; v2.1 backups restore correctly.

**Platform requirement:** `react-native-quick-crypto` is a JSI module. Calling `encryptBackup()` or `decryptBackup()` inside Expo Go throws a clear runtime error. Backup requires an EAS Build or local development build.

**The encryption routine:**
```typescript
export async function encryptBackup(passphrase: string): Promise<string> {
  requireNativeCrypto();

  // Dynamic JSI import — throws clearly on Expo Go or web
  const { pbkdf2Sync, randomBytes, createCipheriv, createHmac, Buffer } =
    await import('react-native-quick-crypto');

  // Collect all vault data
  const [moodLogs, journalEntries, circadianLog, sobrietyStart, name, substance] =
    await Promise.all([
      Sanctuary.getMoodLogs(5000),
      Sanctuary.getJournalEntries(2000),
      Sanctuary.getCircadianLog(365),
      Surface.getSobrietyStart(),
      Surface.getName(),
      Surface.getSubstance(),
    ]);

  const payload: BackupPayload = {
    v: BACKUP_VERSION,
    exportedAt: Date.now(),
    sanctuary: { moodLogs, journalEntries, circadianLog },
    surface: { sobrietyStart, name, substance },
  };

  const plaintext = Buffer.from(JSON.stringify(payload), 'utf8');

  // OS CSPRNG — /dev/urandom (Android), SecureRandom (iOS)
  const salt = randomBytes(32);
  const iv   = randomBytes(16);

  // Native PBKDF2-SHA256 via OpenSSL / CommonCrypto
  // Key derivation state runs entirely in native memory — does not expand through JS heap
  const key = pbkdf2Sync(passphrase, salt, 100_000, 32, 'sha256');

  // Native AES-256-CBC
  const cipher = createCipheriv('aes-256-cbc', key, iv);
  const ciphertext = Buffer.concat([cipher.update(plaintext), cipher.final()] as any[]);

  // Encrypt-then-MAC — HMAC-SHA256 over ciphertext (not plaintext)
  const mac = createHmac('sha256', key).update(ciphertext).digest();

  const blob = {
    v: BACKUP_VERSION,
    alg: 'AES-256-CBC-PBKDF2-SHA256',
    iters: 100_000,
    salt: salt.toString('hex'),
    iv:   iv.toString('hex'),
    mac:  mac.toString('hex'),
    data: ciphertext.toString('base64'),
  };

  return JSON.stringify(blob);
}
```

**The decryption routine — constant-time HMAC check before decrypt:**
```typescript
export async function decryptBackup(blobStr: string, passphrase: string): Promise<BackupPayload> {
  requireNativeCrypto();

  const { pbkdf2Sync, createDecipheriv, createHmac, timingSafeEqual, Buffer } =
    await import('react-native-quick-crypto');

  const blob = JSON.parse(blobStr) as EncryptedBlob;

  const salt = Buffer.from(blob.salt, 'hex');
  const iv   = Buffer.from(blob.iv, 'hex');

  const key = pbkdf2Sync(passphrase, salt, blob.iters, 32, 'sha256');

  // Verify MAC — constant-time comparison eliminates timing side-channel
  const ciphertext  = Buffer.from(blob.data, 'base64');
  const expectedMac = createHmac('sha256', key).update(ciphertext).digest();
  const actualMac   = Buffer.from(blob.mac, 'hex');

  if (!timingSafeEqual(expectedMac, actualMac)) {
    throw new Error('Incorrect passphrase or corrupt backup');
  }

  // Decrypt only if MAC passes
  const decipher = createDecipheriv('aes-256-cbc', key, iv);
  const plaintext = Buffer.concat([decipher.update(ciphertext), decipher.final()] as any[]);

  return JSON.parse(plaintext.toString('utf8')) as BackupPayload;
}
```

**The restore routine — wipes before importing:**
```typescript
export async function restorePayload(payload: BackupPayload): Promise<void> {
  await Sanctuary.burnAll();  // wipe existing data first

  if (payload.surface.sobrietyStart) await Surface.setSobrietyStart(payload.surface.sobrietyStart);
  if (payload.surface.name)           await Surface.setName(payload.surface.name);
  if (payload.surface.substance)      await Surface.setSubstance(payload.surface.substance);
  await Surface.setOnboarded();

  // Restore oldest-first to preserve chronological ordering
  for (const log of [...payload.sanctuary.moodLogs].reverse()) {
    await Sanctuary.saveMoodLog({ ts: log.ts, moodScore: log.moodScore,
      context: log.context, circadianPhase: log.circadianPhase,
      restlessness: log.restlessness, correction: log.correction });
  }
  for (const entry of [...payload.sanctuary.journalEntries].reverse()) {
    await Sanctuary.saveJournalEntry({ ts: entry.ts, body: entry.body,
      moodScore: entry.moodScore, circadianPhase: entry.circadianPhase });
  }
}
```

**What to verify:**
- `encryptBackup` makes no `fetch()` call. The entire function is local.
- `decryptBackup` makes no `fetch()` call. The entire function is local.
- MAC check (`timingSafeEqual(expectedMac, actualMac)`) is constant-time and happens before `createDecipheriv`. Wrong passphrase → wrong key → wrong HMAC → rejected before any decryption.
- `Sanctuary.burnAll()` is called before any restore data is written — clean slate.
- The passphrase is never stored, logged, or sent anywhere. It is used to derive a key, then exits scope.
- Search `backup.ts` for `setItem`, `setItemAsync`, `AsyncStorage`, `SecureStore` — zero results.
