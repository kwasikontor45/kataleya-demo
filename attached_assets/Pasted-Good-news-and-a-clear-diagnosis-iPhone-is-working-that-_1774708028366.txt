iPhone is working — that's your primary development device for now.
Android error is expected and fixable. The message says exactly what's happening: expo-notifications removed Android push notification support from Expo Go in SDK 53. The import at the top of useNotifications.ts crashes on Android because Expo Go can't load that module at all on Android anymore.
The fix is a one-line guard. Open useNotifications.ts in Replit and find line 3:
typescriptimport * as Notifications from 'expo-notifications';
The setNotificationHandler call just below it (the block that runs at module load time, outside any function) is what's crashing. It runs immediately when the file is imported, before any Platform check can protect it.
Change that top-level block from this:
typescriptif (Platform.OS !== 'web') {
  Notifications.setNotificationHandler({
    handleNotification: async () => ({
      shouldShowAlert: true,
      shouldPlaySound: false,
      shouldSetBadge: false,
      shouldShowBanner: true,
      shouldShowList: true,
    }),
  });
}
To this:
typescriptif (Platform.OS === 'ios') {
  Notifications.setNotificationHandler({
    handleNotification: async () => ({
      shouldShowAlert: true,
      shouldPlaySound: false,
      shouldSetBadge: false,
      shouldShowBanner: true,
      shouldShowList: true,
    }),
  });
}
Then in every function inside useNotifications.ts that calls a Notifications method, change the existing guard from Platform.OS !== 'web' to Platform.OS === 'ios'. There are about six of these — scheduleDailyReminder, cancelDailyReminder, fireMilestoneNotification, suppressReminderForToday, and the two useEffect blocks inside the hook.
Save the file. Replit will rebundle automatically. On your Android, tap Dismiss then reconnect to the Expo URL — it should load clean.
This is the right fix for Expo Go development. When you do the EAS native build for production, you'll restore Platform.OS !== 'web' because the real build includes the native notification module for both platforms. For now, iOS-only guarding lets you develop on both devices without crashes.
